# Безопасность публичной версии

Перед публикацией проекта на GitHub из рабочей версии должны быть удалены:

- BOT_TOKEN;
- реальные chat_id;
- реальные message_thread_id;
- телефоны клиентов;
- имена клиентов;
- адреса клиентов;
- рабочие базы SQLite;
- логи с персональными данными;
- IP-адреса серверов;
- внутренние названия групп, если они раскрывают рабочую инфраструктуру.

Вместо рабочих данных нужно использовать:

- .env.example;
- тестовые примеры;
- обезличенные сообщения;
- демонстрационные категории;
- пустую или тестовую SQLite-базу.
