# Архитектура

## Упрощённая схема

1. Telegram-группы с заявками.
2. Бот получает новые и отредактированные сообщения.
3. Сообщения проходят базовую обработку и классификацию.
4. Данные сохраняются в SQLite.
5. По командам или расписанию бот формирует отчёты.
6. Отчёты отправляются в Telegram-группу или тему.

## Основные компоненты

- Telegram Bot API;
- обработчик сообщений;
- SQLite-база;
- модуль классификации заявок;
- модуль отчётов;
- планировщик ежедневных отчётов;
- systemd-сервис для постоянной работы.

## Почему SQLite

SQLite подходит для такого проекта, потому что:

- не требует отдельного сервера базы данных;
- достаточно надёжен для небольших и средних внутренних инструментов;
- легко бэкапится;
- подходит для Telegram-ботов и внутренних сервисов.
