# Telegram Stats Bot

Telegram Stats Bot — бот для автоматической статистики заявок, отчётов по категориям, источникам и периодам.

## Описание проекта

Проект создан для сервисной компании, где заявки поступают в рабочие Telegram-группы. Бот собирает данные по заявкам, группирует их по категориям и источникам, формирует отчёты за день, неделю, месяц и выбранную дату.

Решение помогает руководителям и сотрудникам быстро видеть загрузку, структуру заявок, проблемные направления и динамику работы без ручного подсчёта.

## Что умеет бот

- собирать заявки из Telegram-групп;
- учитывать новые и отредактированные сообщения;
- распределять заявки по категориям;
- считать заявки по источникам;
- формировать отчёты за сегодня, вчера, неделю, месяц и конкретную дату;
- показывать проблемные или некорректно оформленные заявки;
- отправлять ежедневные отчёты в Telegram;
- работать как постоянный systemd-сервис на Linux/VPS.

## Примеры команд

- /today — отчёт за сегодня;
- /yesterday — отчёт за вчера;
- /date — отчёт за выбранную дату;
- /week — отчёт за неделю;
- /month — отчёт за месяц;
- /bad — проблемные заявки;
- /ping — проверка работы бота.

## Что показывает проект

- опыт автоматизации отчётности;
- работу с Telegram Bot API;
- работу с SQLite;
- настройку Linux/VPS и systemd-сервисов;
- сбор и агрегацию данных;
- создание инструментов для руководителей и операционной команды;
- понимание сервисного бизнеса, заявок, источников и категорий.

## Используемые технологии

- Python;
- Telegram Bot API;
- SQLite;

- systemd;

- регулярные отчёты;
- обработка сообщений Telegram.

## Роль в проекте

Проект был создан и внедрён под реальные рабочие процессы: постановка задачи, логика отчётов, категории, источники, команды, проверка работы, сопровождение и доработка под потребности офиса.

## Безопасность публичной версии

В этом репозитории не публикуются:

- токены Telegram-ботов;
- ID рабочих групп и тем;
- реальные заявки;
- телефоны и имена клиентов;
- внутренние данные компании;
- базы SQLite с рабочими данными;
- IP-адреса и доступы к серверам.

Для примера используется только обезличенная структура проекта и документация.

## Статус

Проект использовался как рабочий инструмент для автоматической статистики заявок и управленческой отчётности.
